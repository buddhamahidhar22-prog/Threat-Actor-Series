# 🛡️ Threat Actor Series

Welcome to my **Threat Actor Series**: a monthly cybersecurity research project where I study real-world threat actors from a **Blue Team** and **Defender's** perspective.

Rather than simply documenting who these groups are, this series focuses on understanding **how they operate**, **why their attacks succeed**, and **how defenders can detect, investigate, and respond to them** using modern security practices and Microsoft security technologies.



# 🎯 Project Goal

This repository documents my journey toward becoming a Security Engineer with a focus on:

- Threat Intelligence
- Detection Engineering
- Incident Response
- Identity Security
- Microsoft Security
- Security Operations (SOC)

Every report is written as an opportunity to improve my investigation mindset while building practical knowledge beyond certifications.


# 📚 What Each Report Contains

Each Threat Actor report follows a consistent structure:

- Executive Summary
- Threat Actor Profile
- Timeline of Operations
- Victimology
- Initial Access Techniques
- Attack Chain Analysis
- MITRE ATT&CK Mapping
- Detection Opportunities
- Microsoft Security Perspective
- Incident Response Walkthrough
- Lessons Learned
- Personal Analysis
- References

# 💡 Why I Started This Project

I believe the best way to learn cybersecurity is by understanding how real attackers think.

Instead of memorizing techniques for certification exams, I wanted to study actual threat actors, analyze their attack chains, map their techniques to MITRE ATT&CK, and document how I would defend against them if I were responding to the incident.

This repository is both a learning journal and a technical portfolio that reflects my growth as I continue developing my skills in cybersecurity.
